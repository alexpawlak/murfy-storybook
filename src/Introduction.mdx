# Murfy Design System

This Storybook is the single source of truth for the **murfy.fr redesign** — a design system built for
designers and developers to document, test, and iterate on every UI component before it ships to production.

## What we're building

Murfy is expanding beyond its core **électroménager repair** activity into two new verticals:

| Activity | Accent |
|---|---|
| ⚙️ Électroménager (réparation) | Default — brand teal / pink |
| 🔥 Chauffage | Fuschia |
| ☀️ Solaire | Yellow |
| 🔮 Premium / Highlight | Violet |

Each activity gets its own **accent color** applied via the `data-accent` attribute. The same component — a
button, a hero section, a CTA band — adapts its highlight colors automatically depending on which vertical
it's used in. Switch the **Accent** dropdown in the toolbar above to see this in action.

## How the token system works

```
tokens.json  →  src/styles/tokens.css  →  tailwind.config.js
```

- **Primitives** — raw color values (`--pink-500`, `--yellow-500`, …)
- **Themes** — semantic roles per theme (`--btn-bg`, `--text`, `--bg`, …) applied via
  `data-theme="light|dark|light-pink|pink-mist|pink-vivid"`
- **Accents** — per-activity overrides (`--accent-btn-bg`, `--accent-highlight`, …) applied via
  `data-accent="default|violet|yellow|fuschia"`

The **Theme** and **Accent** toolbar dropdowns at the top of every story let you preview any combination live
without touching code.

## Content sourcing

Some stories pull real content from the live **murfy.fr** site (nav links, service names, copy). When building
new components, check the live site first to use accurate labels, categories, and CTAs rather than
placeholder text.

## Atomic structure

Components follow atomic design:

- **Foundation** — tokens, colors, typography, spacing, theme × accent matrix
- **Atoms** — Button, Input, Tag, Typography
- **Molecules** — NavBar, ServiceCard, PostalCodeForm, AccordionItem
- **Organisms** — Hero, Header, Footer, FAQSection, CTABand, BenefitsGrid
- **Templates** — slot-based page shells
- **Pages** — fully composed page stories (Homepage, Solaire, Chauffage)
