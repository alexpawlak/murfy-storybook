Usage Rules
1
One Primary per visual section. Never stack two Primary buttons side by side without clear hierarchy. The Primary draws the eye — diluting it with two equals kills conversion.
2
Primary + Secondary pairing. When two actions coexist (e.g. "Prendre RDV" + "En savoir plus"), use Primary for the goal action and Secondary for the escape. Never use Accent as a secondary.
3
Accent overrides vertical identity. Use Accent only when the parent section has data-accent set. Don't mix accents on the same page section — one vertical = one accent.
4
Size matches context. Large (56px) for hero sections. Medium (48px) for cards, forms, nav CTA. Small (40px) for inline, tags, and filters. Never use Large inside a card.
5
Icon is mandatory on Primary & Accent. The arrow circle reinforces action affordance. It can be hidden on Secondary when space is very tight (mobile navigation only).
6
Disabled ≠ Hidden. Show disabled state when an action is temporarily unavailable (form validation). If an action will never be available in context, hide the button entirely.
Do / Don't
✓ Do
Clear hierarchy: one Primary for the main CTA, one Secondary for the alternative action.
✗ Don't
Two competing Primary-weight buttons with different accent colors — confuses hierarchy and vertical identity.
✓ Do
Use Small (40px) inside nav or compact cards — right-sized for the context.
✗ Don't
Large (56px) button in the navigation bar — disproportionate and competes with content.
